Step 0 Prerequisites

Before you begin, make sure you have the following ready:

  • A VPS or server with Coolify installed (or access to Coolify Cloud)
  • A domain name pointed to your server's IP address
  • Firebase project with Firestore and Authentication enabled
  • Firebase Admin SDK credentials (service account JSON)
  • GitHub account with access to the SmartQRCode repository

Step 1 Set Up Coolify Server

Access Coolify Dashboard

  1. Open your browser and go to your Coolify URL (e.g., https://coolify.yourdomain.com)
  2. Log in with your admin credentials
  3. You should see the Coolify dashboard
üì∏ Coolify Dashboard - Main view showing servers and projects

Verify Server Connection

  1. Click on "Servers" in the left sidebar
  2. Ensure your server shows a green status
  3. If not connected, click on the server and check the connection settings

Step 2 Create New Project in Coolify

  1. Click "Projects" in the left sidebar
  2. Click the "+ Add" button (top right)
  3. Enter project details:
    • Name: SmartQRCode
    • Description: QR Code Generator Application
  4. Click "Save"

Create Production Environment

  1. Inside your new project, click "+ New"
  2. Select "Production" as the environment name
  3. Click "Create"
üí°
Tip: You can create multiple environments (staging, development) for testing before production.

Step 3 Connect GitHub Repository

Add New Resource

  1. Inside your Production environment, click "+ New"
  2. Select "Public Repository" (or Private if you've connected GitHub)

Configure Repository

Repository URL: https://github.com/Homeboy20/smartqrcode.git
Branch: master
Build Pack: Nixpacks (recommended) or Dockerfile
  1. Enter the repository URL: https://github.com/Homeboy20/smartqrcode.git
  2. Set branch to: master
  3. Select "Nixpacks" as the build pack
  4. Click "Continue"

This is the most important step. You need to add all required environment variables for the application to work.

Navigate to Environment Variables

  1. In your application settings, click "Environment Variables" tab
  2. Click "+ Add" or use bulk edit mode

Required Environment Variables

Copy and paste these variables, replacing the placeholder values with your actual credentials:

IMPORTANT: FIREBASE_PRIVATE_KEY Format
  • Keep the \n characters - don't convert to actual newlines
  • Wrap the entire value in double quotes
  • Copy directly from the downloaded JSON file's private_key field

Step 5 Configure Build Settings

General Settings

  1. Go to the "General" tab in your application
  2. Configure these settings:
    • Port: 3000
    • Build Command: npm run build
    • Start Command: npm start
    • Install Command: npm ci

If Using Dockerfile

If you selected Dockerfile as build pack:

  • Dockerfile Location: Dockerfile (root)
  • Docker Compose: Leave empty (or use docker-compose.yml for compose mode)

Health Check (Recommended)

  1. Go to "Health Check" tab
  2. Enable health check
  3. Set path to: /api/health or /
  4. Interval: 30 seconds

Step 7 Set Up Domain & SSL

Configure Domain

  1. Go to "Settings" or "Domains" tab
  2. Add your domain: your-domain.com
  3. Optionally add: www.your-domain.com

DNS Configuration

In your domain registrar (Namecheap, Cloudflare, GoDaddy, etc.):

# Add these DNS records:

Type: A
Name: @
Value: YOUR_SERVER_IP
TTL: 3600

Type: A  
Name: www
Value: YOUR_SERVER_IP
TTL: 3600

# Or use CNAME if using Cloudflare proxy:
Type: CNAME
Name: www
Value: your-domain.com

Enable SSL (HTTPS)

  1. In Coolify, SSL is usually auto-configured via Let's Encrypt
  2. Make sure "Generate SSL" or "Let's Encrypt" is enabled
  3. Wait a few minutes for certificate generation
  4. Your site should now be accessible via https://

DNS Propagation
DNS changes can take up to 48 hours to propagate globally, but usually work within 15-30 minutes.

Step 9 Post-Deployment Security

üîí
CRITICAL SECURITY STEP
Remove the ADMIN_SETUP_SECRET immediately after creating your admin account!

Remove Admin Setup Secret

  1. Go back to Coolify ‚Üí Your Application ‚Üí Environment Variables
  2. Find ADMIN_SETUP_SECRET
  3. Delete it or set it to empty
  4. Redeploy the application

Security Checklist

  • ADMIN_SETUP_SECRET removed from environment
  • SSL certificate is active (HTTPS working)
  • Firebase security rules are properly configured
  • All sensitive variables are marked as "Secret" in Coolify
  • Admin account has a strong password

Firebase Security Rules

Make sure your Firestore rules are secure. The application includes default rules in firestore.rules.

// Deploy rules from Firebase Console or CLI:
firebase deploy --only firestore:rules

? Troubleshooting

‚ùå Build Fails

Common causes:
  • Missing environment variables
  • Invalid FIREBASE_PRIVATE_KEY format
  • Node.js version mismatch
Solution: Check build logs for specific errors. Ensure all required env vars are set.

‚ùå 500 Internal Server Error

Common causes:
  • Firebase credentials not set correctly
  • Private key has wrong format (missing \n or quotes)
  • Firestore not enabled in Firebase
Solution: Check application logs in Coolify. Verify Firebase setup.

‚ùå Admin Setup Page Shows Error

Common causes:
  • ADMIN_SETUP_SECRET not set
  • Secret doesn't match what you entered
  • Not logged in when trying to set up
Solution: Make sure you're logged in first, then enter the exact secret.

‚ùå SSL Certificate Not Working

Common causes:
  • DNS not pointing to server
  • Port 80/443 blocked by firewall
  • Domain verification failed
Solution: Check DNS propagation at dnschecker.org. Ensure ports are open.

üìù View Logs

To debug issues, check the application logs:

  1. In Coolify, go to your application
  2. Click "Logs" tab
  3. Look for error messages

üîÑ Force Rebuild

Sometimes a fresh build fixes issues:

  1. Go to "Deployments" tab
  2. Click "Rebuild" or "Deploy"
  3. Check "Force rebuild without cache" if available
